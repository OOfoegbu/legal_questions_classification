INTRODUCTION¶

This project would make use of data collected from a free source website. The website contains questions asked by users from around the country, and answers are provided by attorneys. The entries are grouped into 18 legal forums.

The different legal forums are: 1) Auto accident, 2) Bankruptcy, 3) Business, 4) Collections & Debt, 5) Consumer & Lemon, 6) Child Custody, 7) Criminal Defense, 8) Divorce, 9) DUI & DWI, 10) Employment & Labor, 11) Immigration, 12) Insurance, 13) Landlord & Tenant, 14) Medical Malpractice, 15) Personal Injury, 16) Real Estate, 17)Traffic, and 18) Wills,Trust & Probate.

The data collected from the website are:

  • Question title - A brief description of the question.
  • Question detail - Detailed description of the question.
  • State - The US state the user resides. These include DC, Puerto Rico, and US Virgin Islands.
  • Date answered - The estimated date answered. This is calculated from the date it was retrieved from the website.
  • Attorney - The name of the attorney or law firm who provided the first answer to the question.

There are 126597 questions in the dataset.

In a separate report I discuss exploratory finding on the dataset. In this report, I will be building a classification model that takes the question title and detail, and determines what legal forum it belongs to.

Features Engineering¶

I used regular expressions to split the text in the question description into unigrams while removing punctuations except apostrophe and hypens. Then all the unigrams were changed to lower case. Changing to lowercase will avoid the model counting words like 'Process' and 'process' as different words.

In [9]:
## Change text to tokens by splitting word, removing punctuations except apostrophe and hyphens
train_tokens = [[] for i in range(len(train_doc))]

for i in range(len(train_doc)):
    rgx = re.compile("(\w[\w'|-]*\w|\w)")
    train_tokens[i] = rgx.findall(train_doc[i])
    
## Change tokens to lower case
train_tokens_lower = [[] for i in range(len(train_tokens))]

for i in range(len(train_tokens)):
    for w in train_tokens[i]:
        train_tokens_lower[i].append(w.lower())
        
## create dataframe 
train_tokens_data = pd.DataFrame()

train_tokens_data['category'] = train_target
train_tokens_data['token'] = train_tokens_lower

To further clean the data, I created a custom stop words list. The custom stop words list consists of words common to all legal categories, as well as words from the english stopwords dictionary. These words will not be useful as features in the classfication model as they are common to all categories, so they were removed from the unigrams.

Exploratory Analysis - Similarity between legal categories¶

I made use of the intersection of unigrams and bigrams between pairs of the various legal categories to have an idea of which categories a classification model may find challenging to distinguish. This would help understand the shortcomings of the classfication model.

## defined function to create set of unique unigrams and bigrams
def unique_unigrams_bigrams(data):
    bigrams = data['bigrams']
    unigrams = data['unigrams']
    
    bigrams = list(itertools.chain.from_iterable(bigrams))    
    unigrams = list(itertools.chain.from_iterable(unigrams))
    unigrams_bigrams = [unigrams,bigrams]
    unigrams_bigrams = list(itertools.chain.from_iterable(unigrams_bigrams))
    unique_unigrams_bigrams = set(unigrams_bigrams)
    
    return unique_unigrams_bigrams

Build Text Classification Model¶

I built 5 text classification model. They are:

  • Logistic Regression Model
  • Stochastic Gradient Descent Classifier Model
  • Multinomial Naive Bayes Classification Model
  • Linear Support Vector Classification Model
  • Random Forest Classification Model

I made use of unigrams and bigrams for features in the classification models. Then I generated a term frequency matrix. Using the term frequency matrix and tuning the hyper-parameters, I was able to build the classification models except for the random forest model.

In order to improve the accuracy of the random forest model while preventing overfitting, I had to generate a term frequency inverse document frequency matrix. This set weights on the features based on document frequency.

The learning curves for the various models are shown below.

def plot_confusion_matrix(cm, classes,
                          normalize=False,
                          title='Confusion matrix',
                          cmap=plt.cm.Blues):
    """
    This function plots the confusion matrix.
    Normalization can be applied by setting `normalize=True`.
    """
    plt.imshow(cm, interpolation='nearest', cmap=cmap)
    plt.title(title)
    plt.colorbar()
    tick_marks = np.arange(len(classes))
    plt.xticks(tick_marks, classes, rotation=45)
    plt.yticks(tick_marks, classes)
    
    thresh = cm.max() / 2.
    for i, j in itertools.product(range(cm.shape[0]), range(cm.shape[1])):
        plt.text(j, i, cm[i, j],
                 horizontalalignment="center",
                 color="white" if cm[i, j] > thresh else "black")

   
    plt.tight_layout()
    plt.ylabel('True label')
    plt.xlabel('Predicted label')

# Compute confusion matrix
